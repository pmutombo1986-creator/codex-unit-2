# Level 04 — Initialize boolean variables with question-word names

## Objective

Declare and initialize 10 boolean variables on the same line using descriptive camelCase names that begin with `is`, `has`, `can`, or `should`.

## Benefits

- Reinforces naming conventions for booleans.
- Helps distinguish boolean semantics from other types.

## Complete these tasks

- Add 10 boolean variables initialized on the same line with `true` or `false`.
- Use names like `isLoggedIn`, `hasSubscription`.
- Export one of the boolean variables as the `default` export so unit tests can inspect a submitted value.
- Write each line from scratch; do NOT copy & paste any code. Typing each line reinforces syntax memory and debugging skills.

## Hints

- Booleans are literal: `let isReady = true;` or `let hasItems = false;`.

## More information

- Start boolean variable names with a question-style prefix for clarity.

## Usage tips

- Keep names short and descriptive.

## Example

```js
let isLoggedIn = false;
```
